# Mismatch Types

Use these labels when reporting contract review findings.

## Confirmed Alignment

- contract and consumer expectation agree

## Ambiguous Contract

- the available docs or schema do not prove the expected behavior clearly

## Stale Docs

- written docs lag behind stronger evidence such as schema or live payloads

## Consumer Drift

- the frontend or plan assumes a shape or rule not backed by contract evidence

## Missing Support

- the desired behavior is not supported by the provider yet
