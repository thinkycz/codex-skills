# Slice Boundaries

Use this reference to choose useful execution slices.

## Prefer Slices That

- produce a verifiable outcome
- unblock a clear next step
- align with one behavior, integration, or risk area
- fit within one coherent implementation pass

## Avoid Slices That

- are just arbitrary file groupings
- mix unrelated dependencies
- cannot be verified independently
- are so small they create tracking overhead without clarity
