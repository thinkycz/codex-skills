# Capability Selection

## Stronger Capability

Use for:

- architecture
- planning
- integration
- debugging
- high-risk code
- final review

## Cheaper Capability

Use for:

- repetitive edits
- narrow code tasks
- tests
- mechanical migrations
- low-risk support work

## Escalation Rule

If a cheaper worker repeatedly stalls or produces weak results, escalate the slice instead of retrying blindly.
