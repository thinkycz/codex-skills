# Source Quality

Use the strongest available source for the question at hand.

## Preference Order

1. local repo truth
   - source code
   - tests
   - configs
   - schemas
   - docs committed with the project
2. primary external sources
   - official documentation
   - official vendor pages
   - source repositories
   - standards bodies
3. high-quality secondary sources
   - reputable technical writeups that clearly cite primary sources
4. weak or noisy sources
   - forum summaries
   - SEO content
   - unsourced blog posts

## Rules

- Prefer primary sources whenever external information could have changed.
- If two strong sources disagree, surface the conflict instead of picking one silently.
- If only weak sources exist, say so and lower the confidence of the conclusion.
- Use the minimum number of sources needed to resolve the decision safely.
