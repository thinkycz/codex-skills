# Brainstorming Defaults

Use `product-brainstorming` when:
- product intent is still ambiguous
- the request is user-facing and likely to create rework if assumptions are wrong
- there are meaningful solution tradeoffs worth comparing

Do not use it when:
- the design is already fixed enough to write a PRD or implement directly
- the request is a narrow coding change with no product-direction ambiguity
- the next step is already clearly owned by planning or execution
