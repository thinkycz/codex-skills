# Context Budget Heuristics

Use this when reviewing whether a skill is lean enough to keep loading productively.

## Signs A Skill Is Lean

- the trigger is easy to understand
- the owner boundary is obvious
- core workflow fits comfortably in the main file
- references hold the reusable depth

## Signs A Skill Is Bloated

- repeated checklists also appear in neighboring skills
- long prose explains philosophy more than execution
- edge cases dominate the core file
- examples or reference material crowd out the workflow

## Recommended Fixes

- move reusable detail into `references/`
- trim repeated wording across sibling skills
- narrow the trigger boundary
- split only when two distinct workflows are truly competing inside one file
