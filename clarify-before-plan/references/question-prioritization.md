# Question Prioritization

Ask the fewest questions that reduce the most downstream risk.

## Priority Order

1. questions that change architecture or system boundaries
2. questions that change data model or state behavior
3. questions that change acceptance criteria or test strategy
4. questions that change critical UX paths
5. questions that improve completeness but can still be deferred

## Good Question Shape

- short
- specific
- answerable in one sentence or a small set of choices
- tied to an implementation consequence

## Avoid

- broad product-discovery prompts
- duplicate questions that probe the same uncertainty
- questions whose answers do not affect planning or validation
