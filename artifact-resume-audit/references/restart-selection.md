# Restart Selection

Choose the smallest safe restart point.

## Good Restart Points

- one active slice with clear blocker status
- one verification gap that must be closed next
- one domain phase whose artifacts are still trustworthy

## Bad Restart Points

- restarting the whole roadmap because one tracker is stale
- jumping into coding while contradictions remain unresolved
- trusting the newest artifact automatically without checking evidence

## Rule

- Resume where confidence is highest and rediscovery cost is lowest.
