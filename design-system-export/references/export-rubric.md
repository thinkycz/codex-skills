# Export Rubric

Use this when deciding what should become part of the reusable design system.

## Good System Candidates

- stable semantic colors
- typography hierarchy used across multiple surfaces
- shared spacing and density rules
- repeated component states and interaction behavior
- layout principles that apply across the product
- clear anti-patterns the product should consistently avoid

## Bad System Candidates

- one-off campaign visuals
- screen-specific exceptions
- unresolved experiments
- styling that only works in a single context

## Rule

- If the pattern would not help build the next screen consistently, it probably does not belong in the system export yet.
