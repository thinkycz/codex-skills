# Prioritization

Prioritize redesign moves by impact, confidence, and risk.

## Fast Wins

- weak hierarchy fixed by type and spacing improvements
- placeholder or incomplete states replaced with real UX states
- obvious generic card or hero patterns replaced with stronger structure

## Structural Changes

- navigation or layout architecture changes
- major design-system inconsistency across many screens
- shell, panel, or route/state ownership redesign

## Blocked Items

- work that needs a new design source
- work that needs product decisions first
- work that depends on missing assets, brand rules, or backend support
