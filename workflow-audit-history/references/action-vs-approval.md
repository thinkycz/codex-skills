# Action Versus Approval

Separate execution from authorization.

## Actions

Actions are things that actually happened:

- files edited
- commands run
- tests executed
- tasks delegated
- docs updated

## Approvals

Approvals are decisions that authorized or accepted important moves:

- human approval for a direction change
- lead-agent approval for delegation boundaries
- explicit release or handoff acceptance
- source-of-truth priority decisions

## Why The Distinction Matters

- an action may be executed without being formally approved
- an approval may exist before the action happens
- later audits are clearer when these are not blended together
